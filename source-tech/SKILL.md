# Source Tech

Use for AI/tech/startup/builder ecosystem updates, model/tool releases, developer platforms, and technical product shifts.

Read first:
- `../source-common/references/safety-boundaries.md`
- `../source-common/references/research-contract.md`
- `../source-common/references/stop-rules.md`
- `../source-common/references/source-quality-ladder.md`
- `../source-common/references/citation-and-timestamp-rules.md`

Workflow:
1. Gather official release notes/docs/blogs/repos first.
2. Use HN/blog/social only as discovery or practitioner signal.
3. For model/tool/company claims, verify against primary sources.
4. Preserve source notes and adoption/sentiment signals separately.
5. Send notes to `source-briefing`.

Output:
- tech scope
- official/current sources
- discussion/adoption signals
- unverified claims
- missing evidence
- citation/source link
