# Output Shapes

## Source notes

- source/title
- URL
- date/time
- quality tier
- key claim(s)
- caveat or missing proof

## Brief

- answer/verdict first
- 3-8 evidence-backed findings by default
- for Telegram/scheduled briefs, use card-style blockquotes from the chat brief style ref
- citations inline as labelled links
- confidence/source chips when useful
- missing evidence / uncertainty
- next checks or watchpoints

## Deep evidence map

- question
- answer first
- claims and supporting sources
- contradictions / counter-evidence
- confidence and caveats
- next reading/checks

## Watchlist update

- new/material changes
- changed materially
- no meaningful change, only if useful
- stale sources needing refresh

## Artifact

- brief.md
- report-list.md when useful
- notes.md
- references/sources.md
- references/citation-map.md
- references/rejected.md when useful

If the user asks for a sample artifact, show the sample tree/content only; do not write files unless requested.

Visual prompts or generated visuals are output/presentation, not source evidence.

## Tiny examples

Chat brief: verdict first; 3 cited findings; confidence/caveats; missing evidence.

Artifact handoff: `brief.md`, `references/sources.md`, `references/rejected.md` only when requested.

Negative social: "X says X" stays signal only until primary/current sources verify it.

Keep Telegram/chat output compact unless the user asks for depth.
