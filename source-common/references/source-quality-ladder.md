# Source Quality Ladder

Use the strongest available source for each claim.

## Tiers

- A: official/primary — docs, filings, papers, repos, regulator/company statements, direct datasets.
- B: reputable secondary — wire services, established publications, named expert analysis with evidence.
- C: useful signal — HN, Product Hunt, GitHub trending, credible personal blogs/social accounts.
- D: weak signal — aggregators, anonymous social, repost chains, unsourced screenshots.
- E: reject — no URL/timestamp, obvious slop, fabricated-looking media, contradicted claims.

## Rules

- A source can be primary for what it says, not necessarily proof that the claim is true.
- Social and community sources are discovery/sentiment signals unless independently verified.
- Product Hunt attention, GitHub stars, and HN comments are attention/adoption signals, not quality proof.
- Finance/market claims need separate market-data verification.
- Breaking-news claims need primary or multiple credible current sources, or explicit uncertainty.
