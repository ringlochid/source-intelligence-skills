# Watchlist State Rules

Watchlists are curated source memory, not continuous monitoring.

## Entry fields

- Type
- URL/query
- Priority
- Use for
- Last checked
- Freshness
- Notes / verification rules

## Rules

- keep entries short and human-readable
- prefer live workspace watchlists over templates
- do not add private/authenticated sources unless explicitly requested
- update only relevant files
- use stale `Last checked` as a refresh cue, not as proof
- omit quiet areas from briefs instead of padding
